# 前端最終交付驗收測試計畫

**日期：** 2026-06-30  
**功能文件：** `營建日報系統_討論紀錄.md`、`docs/mvp-scope.md`  
**測試對象：** `index.html` / GitHub Pages 線上版本  
**測試範圍：** 工地主任實機試用前的前端完整交付驗收，包含手機填報、資料保存、PDF 產生、歷史日報與跨頁同步。

> 本測試計畫假設目前仍是純前端 localStorage 版本。Supabase、登入、多使用者同步、照片上傳、下包派工不在本次驗收範圍。

---

## 測試環境

### 必測環境

- [x] 桌面瀏覽器：Chrome 或 Safari，直接開啟 `index.html` 或 GitHub Pages。
- [x] iPhone Safari：使用正式線上網址開啟。
- [x] iPhone 主畫面模式：Safari 開啟網址後「加入主畫面」，再從桌面圖示進入。

### 注意事項

- 不使用 LINE 內建瀏覽器作為正式驗收通過標準；若測試 LINE 內建瀏覽器，只記錄為已知限制。
- PDF 下載與分享必須至少在 iPhone Safari 測一次。
- 每輪測試前可用不同日期或清除 `cdr-` 開頭資料避免互相污染。

### 已修正問題

**Bug：設定頁新增工種後，步驟 2 未立即同步**

- 現象：在設定頁新增工種後，settings 有保存，但當前日報步驟 2 沒有馬上出現該工種。
- 根因：`confirmSettingsCrewAdd()` 只更新設定頁 tag 與 settings 資料，沒有同步更新目前日報的人員 UI。
- 修正：新增工種成功後，若步驟 2 尚未存在同名工種，即時插入一列 0 人的工種列並更新總人數。
- 驗證：`npm test` 通過，`frontend-extended-test` 已覆蓋此情境。
