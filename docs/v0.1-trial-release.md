**日期：** 2026-06-30

**試用對象：** 高鐵綻專案現場人員 / 工地主任  
**使用方式：** iPhone Safari 開啟線上網址，或加入主畫面後從桌面圖示進入  
**主要文件：** `營建日報系統_討論紀錄.md`、`docs/testing/2026-06-30-frontend-final-acceptance-test.md`

---

## 封版結論

v0.1 已達到「現場試用」標準，可以交給工地主任用真實日報流程試用。

本版本的目標不是正式上線，也不是取代公司所有日報流程，而是驗證三件事：

1. 現場人員是否願意每天用手機填日報。
2. 完整填報是否能在收工後 10-15 分鐘內完成。
3. 產出的 PDF 是否足以透過 LINE 分享、列印、留存。

---

## 本版可用功能

### 1. 專案設定

- 專案名稱、公司名稱、開工日、竣工日。
- 工種分類設定與新增。
- 預設材料與機具清單。
- 設定資料會保存在瀏覽器 localStorage。

### 2. 五步日報填寫

- 步驟 1：天氣與施工天數。
- 步驟 2：今日人員。
- 步驟 3：營建材料。
- 步驟 4：機具設備。
- 步驟 5：施工紀錄與備註。

### 3. 施工紀錄

- 新增、編輯、刪除施工紀錄。
- 記錄負責人、施工單位、時段、工時、位置、今日內容、明日內容。
- 支援代工廠商標記與廠商名稱。

### 4. 歷史日報

- 可查看最近日報紀錄。
- 可進入單日詳情查看天氣、人員、材料、機具、施工紀錄與備註。

### 5. PDF 預覽、下載與分享

- PDF 第一頁：日報摘要、專案資訊、天氣、人員統計、材料、機具、簽核欄。
- PDF 第二頁：施工紀錄摘要與明細。
- 支援下載 PDF。
- 支援透過 iPhone Safari 分享 PDF 到 LINE 或系統分享面板。

## 正確使用方式

### 建議方式

1. 使用 iPhone Safari 開啟正式網址。
2. 第一次開啟後，使用 Safari 的「加入主畫面」。
3. 之後從手機桌面圖示進入。
4. 每天填完後產生 PDF，透過 LINE 分享或保存。

### 不建議方式

- 不建議用 LINE 內建瀏覽器作為正式操作入口。
- LINE 內建瀏覽器可用來打開連結，但 localStorage、下載、分享、加入主畫面行為不一定穩定。

---

## 已知限制

### 1. 資料只存在單一裝置

目前資料存於該手機瀏覽器的 localStorage。換手機、清除瀏覽器資料、使用不同瀏覽器，資料不會自動同步。

### 2. 尚未支援登入與權限

目前沒有公司帳號、主任帳號、老闆帳號、下包帳號，也沒有專案權限控管。

### 3. 尚未支援雲端備份

如果手機遺失、瀏覽器資料被清除，已填日報可能無法復原。

### 4. 尚未支援多人同步

不同主任或不同裝置填寫的資料不會互相同步。

### 5. 尚未支援照片

施工照片功能尚未啟用。原因是 localStorage 不適合保存大量圖片，應等後端與檔案儲存規劃完成後再做。

### 6. PDF 格式仍需業務確認

目前 PDF 已可產出與分享，但是否完全符合公司、主管、業主或法規格式，仍需依現場回饋調整。

## 現場試用時要觀察的重點

現場試用已在進行，本文件只列出建議觀察項目，方便回饋收斂：

- 使用者是否知道第一步要做什麼。
- 是否能在 10-15 分鐘內完成每日填報。
- 哪些欄位最常空白、最常填錯、最常不知道怎麼填。
- 施工紀錄欄位是否太多或剛好。
- 材料與機具清單是否符合現場用語。
- PDF 主管是否看得懂、願不願意收。
- 分享到 LINE 後，對方是否能正常開啟 PDF。
- 使用者是否會主動要求「昨天資料帶入」「照片」「多人同步」。

## 不在 v0.1 範圍內

| 功能 | 狀態 | 原因 |
|---|---|---|
| 雲端資料庫 | 下一階段 | 需先建立 Supabase schema 與同步策略 |
| 登入 | 下一階段 | 需 LINE Login / Supabase Auth 串接 |
| 多專案切換 | 下一階段 | 需公司、專案、成員資料模型 |
| 權限分層 | 下一階段 | 需 Row Level Security |
| 下包填報 | 後續版本 | 需派工、審核、角色權限 |
| 照片上傳 | 後續版本 | 需檔案儲存與壓縮策略 |
| App Store 原生 App | 暫不做 | 目前 PWA 成本較低，足夠驗證需求 |

---

## 下一階段建議

v0.1 封版後，下一階段應優先做「後端與資料同步」，而不是繼續堆前端功能。

建議順序：

1. 建立 Supabase 專案。
2. 建立公司、使用者、專案、日報、日報明細、施工紀錄資料表。
3. 設計 Row Level Security 權限規則。
4. 先完成「單一主任登入後雲端保存日報」。
5. 再加入 localStorage 離線草稿與線上同步。
6. 最後才做多角色、多專案、下包填報與照片。
